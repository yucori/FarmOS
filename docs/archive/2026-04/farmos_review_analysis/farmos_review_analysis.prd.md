## 1. 제품 개요

### 1.1 한 줄 요약

FarmOS 쇼핑몰의 농산물 리뷰를 RAG/임베딩 기반으로 자동 분석하여 감성분석, 키워드 추출, 요약, 트렌드 탐지를 제공하고, 농업인 판매자가 판매 전략을 데이터 기반으로 개선할 수 있게 하는 기능.

### 1.2 문제 정의

| 구분 | 내용 |
|------|------|
| **현재 상태** | 리뷰 분석 백엔드/프론트엔드가 구현되어 있으나(archive 96% match), Mock 데이터 50건 기반으로 개발됨. 실제 DB(shop_reviews) 연동 미완료, 자동 배치 스케줄러 미구현, 감성분석 정확도 런타임 검증 미완료 |
| **핵심 문제** | 농산물 판매자가 수백~수천 건의 리뷰를 수동으로 읽고 분석해야 하며, 품질/배송/포장 등 카테고리별 인사이트를 체계적으로 파악하기 어려움 |
| **해결 방향** | 기존 구현물을 실제 운영 수준으로 고도화: DB 연동, 정확도 검증, 스케줄러 구현, UX 개선 |

### 1.3 대상 사용자

| 사용자 | 역할 | 핵심 니즈 |
|--------|------|----------|
| **농업인 판매자** | 자기 상품 리뷰 분석 | 어떤 부분이 호평/불만인지 빠르게 파악, 판매 전략 개선 |
| **관리자/운영자** | 전체 플랫폼 리뷰 모니터링 | 전체 감성 추이 파악, 이상 징후 탐지, PDF 리포트 출력 |
| **농업 컨설턴트** | 데이터 기반 조언 제공 | 여러 농가의 리뷰 트렌드 비교, 키워드 기반 개선 포인트 도출 |

| 항목 | 내용 |

| **조건** | 판매자는 자기 상품 리뷰만, 관리자는 전체 리뷰 조회 가능 |

| 항목 | 내용 |

## 5. 가치 제안 (Value Proposition)

### 5.1 핵심 가치

| 가치 | 설명 |
|------|------|
| **시간 절약** | 수백 건 리뷰 수동 분석 → 1회 클릭으로 자동 분석 (90% 시간 절감) |
| **데이터 기반 의사결정** | "포장이 별로"라는 주관적 느낌 → "포장 관련 부정 리뷰 23%, 전월 대비 5% 증가" 정량 데이터 |
| **조기 경보** | 부정 리뷰 급증, 특정 키워드 급등 시 이상 탐지 알림 → 빠른 대응 가능 |
| **RAG 기반 질의** | "배송 관련 불만" 자연어 검색으로 관련 리뷰 즉시 검색 → 맥락 파악 용이 |

### 5.2 경쟁 분석

| 솔루션 | 특징 | FarmOS 차별점 |
|--------|------|-------------|
| **네이버 스마트스토어 리뷰** | 기본 별점 통계만 제공, NLP 분석 없음 | 감성분석 + 키워드 + 요약 + 트렌드 |
| **쿠팡 리뷰 관리** | 리뷰 목록 조회만, 자동 분석 없음 | RAG 기반 의미 검색 + 자동 배치 분석 |
| **ReviewTrackers (SaaS)** | 월 $49~$399, 영어 중심 | 무료(셀프호스팅), 한국어 농산물 특화 |
| **SentiOne (감성분석 SaaS)** | 월 $300+, 범용 | 농산물 도메인 프롬프트, 비용 1/10 이하 |

### 5.3 시장 규모 (TAM/SAM/SOM)

| 구분 | 규모 | 근거 |
|------|------|------|
| **TAM** | 국내 농산물 온라인 거래 약 8.5조원 (2025) | 농림부 통계, 연 15% 성장 |
| **SAM** | 온라인 판매 농가 약 5만 호 × 리뷰 관리 니즈 | 스마트팜 보급 농가 기준 |
| **SOM** | FarmOS 초기 타깃 500~1,000 농가 | 교육기관/지자체 연계 스마트팜 사용자 |

## 6. 기술 아키텍처

### 6.1 현재 구현 스택

| 구분 | 기술 | 상태 |
|------|------|------|
| 벡터DB | ChromaDB 1.5.5 (내장 임베딩) | 구현 완료 |
| LLM 개발 | Ollama (llama3 로컬) | 구현 완료 |
| LLM 배포 | OpenRouter API | 추상화 구현 완료, 검증 필요 |
| RAG 검색 | ChromaDB query + 메타데이터 필터 | 구현 완료 |
| 감성분석 | LLM 1회 호출 (감성+키워드+요약 동시) | 구현 완료, 정확도 검증 필요 |
| 트렌드 탐지 | 이동평균 + 표준편차 (LLM 불필요) | 구현 완료 |
| PDF 리포트 | fpdf2 2.8.0 | 구현 완료 |
| 프론트엔드 | React + Recharts + useReviewAnalysis 훅 | 구현 완료, UX 고도화 필요 |

### 6.2 AI 모델 비용 전략

> **원칙: 비용 효율성 최우선** (사용자 피드백 반영)

| 환경 | 모델 | 비용 | 용도 |
|------|------|------|------|
| **개발** | Ollama llama3-8b (로컬) | 0원 | 개발/테스트 |
| **배포 옵션 A** | OpenRouter → GPT-4o-mini | ~$0.15/1M tokens | 가장 저렴한 클라우드 옵션 |
| **배포 옵션 B** | RunPod + Ollama (원격) | ~$0.39/hr (GPU) | 대량 배치 처리 시 유리 |
| **임베딩** | ChromaDB 내장 (all-MiniLM-L6-v2) 또는 Ollama nomic-embed-text | 0원 | 벡터 임베딩 |

배치 처리 (20건/1호출) + 결과 캐싱으로 LLM 호출 최소화.
